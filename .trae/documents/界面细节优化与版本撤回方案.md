## 目标概述
- 增强左右/上下分割拖拽能力与抓手样式
- 完善明暗主题下的可读性与语法着色
- 优化交互细节（行级 hover、选中高亮、历史徽标颜色）
- 升级历史逻辑：自动识别“大改动”，并支持一键撤回上一个大版本

## 分割拖拽与抓手
- 上下分割（已实现）：保留现有 `split` 百分比与拖拽逻辑
- 新增左右分割：
  - 新增 `hsplit: number`（默认 55），范围限制 20–80，持久化到 `localStorage('final-json:hsplit')`
  - 将左右两列改为：左列 `style={{ flex: '0 0 hsplit%' }}`，右列 `style={{ flex: '0 0 calc(100% - hsplit% - 8px)' }}`
  - 在左右列中间插入竖向分割条（`h-full w-2 cursor-col-resize`），`onMouseDown` 绑定 `mousemove`/`mouseup` 更新 `hsplit`
- 抓手样式：
  - 分割条中间添加抓手元素（如三条水平线或三个圆点），`w-6 h-[2px] rounded bg-gray-500/70`，hover 时 `bg-gray-300` 并加轻微阴影
  - 明暗模式分别用 `bg-gray-600`/`bg-gray-300` 以提升可见度

## 明暗主题与语法着色
- 新增颜色映射常量 `TOKEN_COLORS[theme]`：
  - 对象键：dark `text-cyan-300`，light `text-cyan-700`
  - 字符串：dark `text-emerald-300`，light `text-emerald-700`
  - 数字：dark `text-sky-300`，light `text-sky-700`
  - 布尔：dark `text-violet-300`，light `text-violet-700`
  - 空值：dark `text-gray-400`，light `text-gray-600`
- 在 `JsonNode` 中：
  - 键名与 `renderValue()` 按 `theme` 动态应用上述类
  - 展开信息文本颜色统一为更高对比度的 `text-gray-300`（dark）/`text-gray-600`（light）
- 白天模式字体：
  - 根容器在 light 模式改为 `text-gray-900 bg-gray-50`，并提升各文本组件的前景色（如编辑区 `text-gray-800`）

## 行级 hover 与选中高亮
- 新增 `selectedPath: Path | null` 状态，点击任一 `JsonNode` 外层容器设置为当前路径
- 高亮逻辑：
  - hover：dark `hover:bg-gray-700/20`，light `hover:bg-gray-200`
  - 选中：在对应行加 `bg-sky-500/10 border border-sky-500/30`，同时在右侧可视化对应节点同步高亮

## 历史徽标颜色修正
- 右上角历史记录的计数徽标从 `bg-red-500` 改为主题色（如 `bg-sky-500 text-white`），hover 不改变前景色，降低突兀感

## 历史逻辑升级与撤回
- 新增 `versions: { id, content, timestamp }[]` 栈（只记录“大改动”的完整快照）
- 定义“大改动”判定：
  - 整份覆盖（检测 `onPaste` 或 `handleJsonChange` 中长度变化比例 `> 0.3`）
  - 删除节点（`updateDataAtPath` 的 `action === 'delete'`）
  - 根级结构变化（根键数量/数组长度变化超过阈值）
- 触发时 `pushVersion(jsonInput)` 并更新 `historyList` 的操作类型标识为“大改动”，其余标记为“普通”
- 新增“撤回”按钮：
  - 放置在右上工具区（历史旁），可点击恢复到 `versions` 栈的上一个快照
  - 恢复时调用现有 `handleJsonChange(snapshot.content)`，并弹出通知
  - 若无上一个版本则禁用按钮
